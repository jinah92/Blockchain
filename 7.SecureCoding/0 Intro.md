# 시큐어코딩 개요



### 가장 많이 일어나는 취약점 ⇒ XSS

- 이유 : 웹페이지 수, 기능이 많아짐에 따라 노출되는 영역이 증가

### 상대적으로  적게/일정하게 발생하는 취약점 ⇒ SQL Injection

- 이유 : 보안 기법이 확실히 정해져 있음
- 왜 여전히 있는지 ? 
⇒ 여전히 안 막는 사이트가 존재

### 스마트폰 앱의 보안 취약점 원인

- 각 서비스 노드(웹, 모바일, db 등)의 분리 부재
- 비즈니스 로직의 분리 ⇒ 처리내용의 동기화 X

### 보안 약점과 보안 취약점

- 실제 보안사고 난 것 ⇒ 보안 취약점

### OWASP TOP 10

- 웹 어플리케이션 개발자들 단체
- 주기적으로 웹 어플리케이션 관련 보안 취약점에 대해 리스팅하여 발표

### (자바) 시큐어코딩의 대상? ⇒ 서비스 컴포넌트 (확장성)

- 자바는 엔터프라이즈 목표로 만들어졌기 때문에, 다음 사항을 고려해서 코딩한다
    1. 보안
    2. 트랜잭션
    3. 영속성 및 확장성
- Tier 분리
    1. **Presentation : 뷰 제공**
    2. **Business** - **(Java EE) 라이브러리**를 통해 요구사항(보안, 트랜잭션 등)에 따라 코드를 삽입
        - Web Application Server (WAS)
